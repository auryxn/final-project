# 🧪 Инструкция по тестированию проекта

Проект состоит из трех частей. Вот как запустить и проверить каждую:

## 1. Инфраструктура (Базы данных)
Убедитесь, что у Вас установлен Docker. В корне проекта выполните:
```bash
docker-compose up -d
```
Это запустит **PostgreSQL** (порт 5432), **MongoDB** (порт 27017) и **Redis** (порт 6379).

---

## 2. Бэкенд (Java Spring Boot)
Перейдите в папку `backend` и запустите приложение:
```bash
cd backend
./mvnw spring-boot:run
```
*   **API доступно по адресу:** `http://localhost:8080/api/traffic/metrics`
*   Бэкенд автоматически подключится к базам в Docker.

---

## 3. Фронтенд (React)
Перейдите в папку `frontend`. Сначала установите зависимости, затем запустите:
```bash
cd frontend
npm install
npm start
```
Дашборд откроется на `http://localhost:3000`. Он начнет запрашивать данные у бэкенда.

---

## 4. Анализ данных (Python)
Если хотите запустить ML-скрипты и генерацию графиков:
```bash
cd analysis
pip install -r requirements.txt  # Если я его еще не создал, установите pandas, seaborn, scikit-learn
python main.py
python stats_and_ml.py
```
Результаты появятся в папке `analysis/plots`.

---

## ✅ Чек-лист проверки:
1.  **БД:** Контейнеры в Docker Desktop горят зеленым.
2.  **API:** При переходе на `http://localhost:8080/api/traffic/metrics` возвращается пустой список `[]` (или данные, если они уже в БД).
3.  **UI:** На странице `http://localhost:3000` видны виджеты и пустые графики, готовые к данным.
